## 🔧 路由机制说明

### 动态路由加载

项目采用**动态路由加载机制**，基于文件路径自动匹配：

```javascript
// 路径规则：/模块名/页面名
// 自动映射到：src/pages/模块名/页面名.jsx

示例：
/cm-system/user  →  src/pages/cm-system/user.jsx
/cm-portal/article/articleList  →  src/pages/cm-portal/article/articleList.jsx
```

### 路由配置文件

- **路由配置**: `src/router/routesConfig.js` - 静态路由映射和动态加载函数
- **路由入口**: `src/router/index.jsx` - 路由守卫和动态路由处理
- **菜单配置**: `src/config/menuConfig.jsx` - 默认菜单配置（后端加载失败时使用）

### 菜单数据流

```
后端API (/getAsyncRoutes)
    ↓
ProLayout组件处理
    ↓
processMenuData() 转换
    ↓
菜单渲染 + 路由导航
    ↓
DynamicRouteHandler 动态加载组件
```

## 📝 开发指南

### 添加新页面

1. **创建页面组件**
   ```bash
   src/pages/模块名/页面名.jsx
   ```

2. **页面组件模板**
   ```jsx
   import React from 'react';
   import CmBasePage from '@components/CmBasePage';

   const PageName = () => {
     return (
       <CmBasePage
         title="页面标题"
         // 其他配置...
       >
         {/* 页面内容 */}
       </CmBasePage>
     );
   };

   export default PageName;
   ```

3. **配置后端菜单**
   - 在后台菜单管理中添加菜单项
   - 设置路径为：`模块名/页面名`
   - 配置权限标识（如需要）

4. **访问路径**
   ```
   http://yourdomain/模块名/页面名
   ```

### 路由规则

- ✅ 支持多级路径：`/cm-portal/article/articleList`
- ✅ 自动组件加载：基于文件路径动态匹配
- ✅ 懒加载：所有页面组件按需加载
- ✅ 路由守卫：未登录自动跳转登录页
- ✅ 404处理：不存在的路径显示"页面开发中"
