# 后端接口返回结构说明文档

## 概述

本文档详细说明了各个后端接口的实际返回结构，基于实际测试和旧项目代码分析。

## 1. 获取验证码接口

### 接口信息
- **URL**: `/code`
- **方法**: GET
- **是否需要Token**: 否

### 实际返回结构

```json
{
    "msg": "操作成功",
    "img": "/9j/4AAQSkZJRgABAQEA...(base64图片数据)",
    "code": 200,
    "captchaEnabled": true,
    "uuid": "41d283a335884086893a10000f01dd92"
}
```

### 关键字段说明

| 字段 | 类型 | 说明 | 位置 |
|------|------|------|------|
| `msg` | String | 响应消息 | 根级别 |
| `img` | String | Base64编码的验证码图片 | 根级别 ⭐ |
| `code` | Number | 状态码（200表示成功） | 根级别 |
| `captchaEnabled` | Boolean | 是否启用验证码 | 根级别 |
| `uuid` | String | 验证码唯一标识 | 根级别 ⭐ |

### ⚠️ 重要说明

**关键字段都在根级别，不在 `data` 对象里！**

### 前端处理代码

```javascript
const fetchCaptcha = async () => {
  const response = await getCodeImage({ timestamp: Date.now() });
  
  // ✅ 正确：直接从根级别获取
  setCaptchaId(response.uuid);
  const imageData = response.img;
  
  // ❌ 错误：不要从 data 对象获取
  // setCaptchaId(response.data?.uuid); // 错误！
};
```

## 2. 登录接口

### 接口信息
- **URL**: `/auth/l_l`
- **方法**: POST
- **Content-Type**: `application/json`
- **是否需要Token**: 否

### 请求参数

```javascript
{
    "username": "admin",
    "password": "123456",
    "authCode": "123456",      // Google验证码 ⭐
    "code": "abc123",           // 图片验证码
    "uuid": "41d283a3..."      // 验证码ID
}
```

### 实际返回结构

**成功响应**:
```json
{
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "msg": "登录成功"
}
```

或者可能包含更多信息:
```json
{
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "user": { ... },
    "roles": [ ... ],
    "permissions": [ ... ],
    "msg": "登录成功"
}
```

**失败响应**:
```json
{
    "msg": "Google验证码错误"
}
```

### 关键字段说明

| 字段 | 类型 | 说明 | 位置 |
|------|------|------|------|
| `token` | String | JWT Token | 根级别 ⭐ |
| `msg` | String | 响应消息 | 根级别 ⭐ |
| `user` | Object | 用户信息（可能返回） | 根级别 |
| `roles` | Array | 角色列表 | 根级别 |
| `permissions` | Array | 权限列表 | 根级别 |

### ⚠️ 重要说明

1. **token在根级别**，不是 `response.data.token`
2. **错误消息在msg字段**，不是 `response.message`
3. Google验证码 `authCode` 是必填字段 ⭐

### 前端处理代码

```javascript
const login = async (loginData) => {
  const response = await apiService.post("/auth/l_l", {
    username: loginData.username,
    password: loginData.password,
    authCode: loginData.authCode,  // ⭐ 必填
    code: loginData.code,
    uuid: loginData.uuid,
  });

  // ✅ 正确：直接从根级别获取token
  const token = response.token;

  // ❌ 错误：不要从 data 对象获取
  // const token = response.data?.token; // 错误！

  if (!token) {
    return { success: false, message: response.msg };
  }

  setToken(token);
};
```

## 3. 获取用户信息接口

### 接口信息
- **URL**: `/system/user/getInfo`
- **方法**: GET
- **是否需要Token**: 是

### 请求头

```
Authorization: Bearer {token}
```

### 实际返回结构

```json
{
    "code": 200,
    "user": {
        "userId": 1,
        "userName": "admin",
        "nickName": "管理员",
        "avatar": "http://example.com/avatar.jpg"
    },
    "roles": [
        {
            "roleId": 1,
            "roleName": "管理员",
            "roleKey": "admin",
            "roleSort": 1
        }
    ],
    "permissions": [
        "system:user:list",
        "system:user:add",
        "system:user:edit"
    ],
    "msg": "操作成功"
}
```

### 关键字段说明

| 字段 | 类型 | 说明 | 位置 |
|------|------|------|------|
| `code` | Number | 状态码（200表示成功） | 根级别 |
| `user` | Object | 用户信息对象 | 根级别 ⭐ |
| `roles` | Array | 角色列表 | 根级别 ⭐ |
| `permissions` | Array | 权限列表 | 根级别 ⭐ |
| `msg` | String | 响应消息 | 根级别 ⭐ |

### ⚠️ 重要说明

**所有数据都在根级别，不在 `data` 对象里！**

### 前端处理代码

```javascript
const userInfoResponse = await apiService.get("/system/user/getInfo");

// ✅ 正确：直接从根级别获取
const { user, roles, permissions } = userInfoResponse;

// ❌ 错误：不要从 data 对象获取
// const { user, roles, permissions } = userInfoResponse.data; // 错误！

setUser(user);
setRoles(roles);
setPermissions(permissions);
```

## 5. 获取路由菜单接口

### 接口信息
- **URL**: `/getRouters`
- **方法**: GET
- **是否需要Token**: 是

### 请求头

```
Authorization: Bearer {token}
```

### 实际返回结构

```json
{
    "code": 200,
    "msg": "操作成功",
    "data": [
        {
            "name": "Platform",
            "path": "/platform",
            "component": "Layout",
            "meta": { "title": "平台数据", "icon": "excel" },
            "children": [
                {
                    "name": "Record",
                    "path": "record",
                    "component": "bussiness/record/index",
                    "meta": { "title": "账变信息" }
                }
            ]
        }
    ]
}
```

### 关键字段说明

| 字段 | 类型 | 说明 | 位置 |
|------|------|------|------|
| `code` | Number | 状态码（200表示成功） | 根级别 |
| `msg` | String | 响应消息 | 根级别 |
| `data` | Array | 路由配置数组 | 根级别 ⭐ |

### ⚠️ 重要说明

**路由数据在 `data` 数组中！** 这是与前面接口不同的地方。

### 前端处理代码

```javascript
const routerResponse = await apiService.get("/getRouters");

// ✅ 正确：从 data 数组获取
const routes = routerResponse.data;

// ❌ 错误：不要直接从根级别获取
// const routes = routerResponse; // 错误！
```

## 接口返回结构总结

| 接口 | 数据在根级别 | 数据在data对象 | 说明 |
|------|-------------|--------------|------|
| **GET /code** | ✅ | ❌ | 验证码数据在根级别 |
| **POST /auth/l_l** | ✅ | ❌ | token在根级别 |
| **GET /system/user/getInfo** | ✅ | ❌ | 用户数据在根级别 |
| **DELETE /auth/logout** | ✅ | ❌ | 状态码在根级别 |
| **GET /getRouters** | ❌ | ✅ | 路由数据在data中 ⭐ |

## 常见错误

### 错误1：访问 data 对象

```javascript
// ❌ 错误示例
const token = response.data.token;
const uuid = response.data.uuid;
const user = response.data.user;

// ✅ 正确示例
const token = response.token;
const uuid = response.uuid;
const user = response.user;
```

### 错误2：错误消息字段

```javascript
// ❌ 错误示例
const message = response.message;

// ✅ 正确示例
const message = response.msg;
```

### 错误3：状态码判断

```javascript
// ⚠️ 需要确认
// 后端返回的状态码可能是 code 或其他字段
if (response.code === 200) {  // 或 response.code === 0
  // 成功处理
}
```
