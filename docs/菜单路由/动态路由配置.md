# 动态路由配置文档

## 路由映射规则

根据后台返回的路由配置，系统使用以下规则动态加载组件：

### 加载规则

```
路由路径 → 组件路径
/bussiness/user → ../pages/bussiness/user.jsx
/deposit/recharge/index → ../pages/deposit/recharge/index.jsx
/system/menu/index → ../pages/system/menu/index.jsx
```

**核心逻辑**：
- 后台返回的 `component` 字段值直接对应 `src/pages/` 下的文件路径
- 例如：`component: "bussiness/user/index"` 对应文件 `src/pages/bussiness/user/index.jsx`
- 路由路径 `/bussiness/user` 会被处理为组件路径 `bussiness/user/index`

### 动态路由加载器

```javascript
// src/router/routesConfig.js
const pageModules = import.meta.glob('../pages/**/*.jsx', { eager: false });

export const loadDynamicComponent = async (path) => {
  const componentPath = `../pages/${path}.jsx`;
  if (pageModules[componentPath]) {
    const module = await pageModules[componentPath]();
    return module.default;
  }
  throw new Error(`Component not found: ${componentPath}`);
};
```

## 开发指南

### 添加新页面

1. 在 `src/pages/` 下创建对应目录和文件
2. 文件名必须与后台返回的 `component` 字段匹配
3. 文件扩展名必须是 `.jsx`
4. 导入必要的组件和API

### 调试路由问题

如果页面无法加载，检查：

1. **文件路径是否正确**
   ```bash
   ls -la src/pages/bussiness/user/index.jsx
   ```

2. **文件扩展名是否正确**
   ```bash
   find src/pages -name "*.jsx" -type f
   ```

3. **组件是否正确导出**
   ```jsx
   export default ComponentName;
   ```

4. **浏览器控制台错误**
   - 打开开发者工具
   - 查看 Console 中的错误信息
   - 检查 Network 中的请求

### 常见问题

**Q: 页面显示"页面开发中"**
A: 文件路径不匹配，请检查后台返回的 `component` 字段与实际文件路径是否一致

**Q: 路由加载404**
A: 检查文件是否存在，路径是否正确

**Q: 组件加载失败**
A: 检查组件是否有语法错误，确保有 `export default`
